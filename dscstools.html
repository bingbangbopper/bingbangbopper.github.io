Editing strings with DSCSTools

Locate your Hundred Line installation. It’s probably:

C:\Program Files (x86)\Steam\steamapps\common\The Hundred Line -Last Defense Academy-
Open it up in Windows Explorer. There should be a folder called
gamedata in here, the HUNDRED_LINE.exe executable itself, and some other stuff.

Extract your DSCSToolsCLI.exe and structures folder directly into this folder, so that DSCSToolsCLI.exe and structures are in the same folder as HUNDRED_LINE.exe and gamedata.

Right click in the Explorer window → Open in Terminal

If you type DSCSToolsCLI.exe you should get the help text:

If it says something like The term 'DSCSToolsCLI.exe' is not recognized... then you either did not unzip the stuff into the correct folder, or you did not open your terminal window to the correct folder.

Keep this terminal window open.

Some notes/explanation of what we’re about to do:

If you look inside of the gamedata folder you will see the game files plus a folder called savedata which stores your save files. Each of these files is like a .zip archive. The files named text00 contain Japanese strings, those named text01 contain English strings, and text02 and text03 are Chinese. Since we’re looking to edit English strings, we only care about the two text01 files.

I don’t know when patch_text01.dx11.mvgl got added. Going by the names, it seems like app_text01.dx11.mvgl is the game’s original set of strings, and patch_text01.dx11.mvgl is a patched subset of strings that the game will prefer to read from. When I did my “shrink ray” edit, I edited the strings in app_text01.dx11.mvgl the first time, not knowing about the patch file. My edits didn’t appear in the game, and I didn’t know why. Then I discovered the patch_text01.dx11.mvgl file, and I realized the game must be reading the strings from that file instead, and editing the strings in there got it to work. So, you should probably prefer to edit the patch strings instead of the app strings.

For the next part I’m going to assume we’re working with the patch file, but these instructions will work with the app file too. Just remember to change the names in the commands.

Run the following commands in the terminal window you opened:

mkdir patch_text01
mkdir csv_text
mkdir csv_message
DSCSToolsCLI.exe --extract .\gamedata\patch_text01.dx11.mvgl .\patch_text01\
DSCSToolsCLI.exe --mbeextract .\patch_text01\text\ .\csv_text\
DSCSToolsCLI.exe --mbeextract .\patch_text01\message\ .\csv_message\

Now at this point there will be a bunch of folders in csv_text and csv_message, each containing one .csv file (easily readable and editable in Notepad or VS Code, but do not edit it in Excel because apparently Excel doesn’t handle the format correctly). In Windows Explorer, look around either the csv_text folder (I believe this is more for UI related strings) or the csv_message folder (I believe this is more for dialogue strings) for the strings you want to edit. For my video where I edited the decision text I had to edit the csv file inside the event_decideselect_message.mbe folder, which contains all the decision strings (I assume).

Suppose you have a string in mind that you are trying to edit. We can find the .csv file that it’s in by using a powershell command. Open the csv_message or csv_text folder in Explorer, and right click and open the folder in Terminal. In Terminal, enter the command sls "enter your string here" -Path .\**\*.csv and Powershell will search for it. For example, from inside the csv_message folder I entered sls "Takumi" -Path .\**\*.csv and it returned all the .csv files inside the csv_message folder that contained the word Takumi. Here’s three examples:

PS [...]\csv_message> sls "takumi" -Path .\**\*.csv
[...]
r00_0_1sttruer_00_d028_m050_010.mbe\Sheet1.csv:41:400,13,"Stand down, Sir Takumi.","pic_voice","Stand down, Sir Takumi.","","","","","","",""
r00_0_1sttruer_00_d028_m070_010.mbe\Sheet1.csv:63:620,14,"So what're you gonna do now, Takumi?","pic_voice","So what're you gonna do now, Takumi?","","","","","","",""
r00_0_1sttruer_00_d028_m110_010.mbe\Sheet1.csv:5:40,11,"Uh... Are you going to fight with us, Takumi?","pic_voice","Uh... Are you going to fight with us, Takumi?","","","","","","",""
[...]

If I wanted to edit that third string, I would find it in csv_message\r00_0_1sttruer_00_d028_m110_010\Sheet1.csv. I can open that file up in Notepad or VS Code and look it over (it contains lines of dialogue from a scene), edit the lines however I want, and save the .csv file, and then when we pack it up back into the mvgl file later it’ll show up in the game.

Once you’ve made your edits, it’s time to pack them back up. We will pack our edited .csv file into its .mbe file, and then pack the .mbe file into the much larger .mvgl file. Open a Terminal window in the same folder you did at the start: the one with gamedata and HUNDRED_LINE.exe along with the folders we made. Now run the following commands. Note: This will overwrite the patch_text01.dx11.mvgl file in your gamedata folder — maybe you want to back it up in case something goes wrong, so that you don’t have to reinstall the entire game.

I’m going off my example above, where I edited csv_message\r00_0_1sttruer_00_d028_m110_010\Sheet1.csv, but you would just change the commands according to the file that you edited:

DSCSToolsCLI.exe --mbepack .\csv_message\r00_0_1sttruer_00_d028_m110_010.mbe\ .\patch_text01\message\r00_0_1sttruer_00_d028_m110_010.mbe
DSCSToolsCLI.exe --pack .\patch_text01\ .\gamedata\patch_text01.dx11.mvgl --disable-crypt
